### A diferença entre `context/` e `application/`

- **`context/`** = a **referência** (fatos verificados, regras, perfis, inteligência). Leia para saber. Não muda muito.
- **`application/`** = o **trabalho** (os textos que vão para o formulário). Escreva aqui. Evolui a cada sessão.

---

## Por onde começar (fluxo de trabalho)

1. **Entender a tese** → `context/proposal/recommended-direction.md`
2. **Conferir as regras** → `context/grant/` (critérios, elegibilidade, orçamento, prazo)
3. **Escrever** → `application/` na ordem 00 → 06
4. **Rastrear** → `application/_tracker/` (decisões, checklist, progresso)

> **Regra de ouro da redação:** todo texto deve soar como **exploração e co-design de parceria e de framework de reconhecimento** — nunca como "pesquisa" nem "produção do MOOC" (gates de eliminação). Sem PI/licença no orçamento. Co-autoria visível dos dois lados.
